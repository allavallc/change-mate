## Goal
Teach the live board to operate in three distinct modes (`public-readonly`, `public-write`, `private`) and wire the Add Story button to the `cm-write` Edge Function instead of raw GitHub PAT + browser localStorage.

## Why
A single mode ("anyone can see, nobody can write without their own PAT") does not fit real projects. Some boards are fully public, some accept anonymous-but-gated contributions, some need to be private. Tying Add Story to a server-side write path also kills the per-user PAT setup friction that today makes the button a non-starter for non-technical users.

## Done when
- `bot-horde-config.json` gains a `board_mode` field: `public-readonly` | `public-write` | `private`
- `public-readonly`: no Add Story button, board visible to anyone
- `public-write`: Add Story button visible, prompts for write key on click, key cached in localStorage
- `private`: board gated behind a write-key prompt on load; no content renders until validated
- Add Story button POSTs to `cm-write` Edge Function with the cached key
- Clear error UX for invalid/revoked keys (not a silent failure)
- Old per-user GitHub PAT path removed from board JS

## Desired output
The live board respects the mode in config, and the Add Story flow no longer touches a user's GitHub PAT.

## Success signals
- Flipping `board_mode` in config produces the expected UX on next build
- Write key prompt appears exactly once per browser (cached in localStorage)
- Add Story round-trips through the Edge Function in under 2s

## Failure signals
- Write keys leaking into analytics / Sentry / URL params
- `private` mode flashing content before the gate renders (FOUC-style leak)
- localStorage cache not invalidating when a key is revoked server-side

## Tests
- Snapshot tests for each of the three modes (readonly, write, private)
- Unit test for write-key prompt flow (first visit, cached visit, revoked key)
- Manual QA checklist for each mode before merge

## Resolution (2026-04-16)
Shipped, but scope collapsed. After building out write-keys + three board modes, we ripped it all out in favor of GitHub-token auth (commit `2af97b4`). The board now follows a single, simpler model:

- No `board_mode` field — board visibility follows the underlying repo's visibility (public repo = public board, private repo = private board).
- Add Story button prompts for a GitHub fine-grained PAT (one-time, cached in localStorage).
- Add Story POSTs to `cm-write` Edge Function with `github_token`. The function verifies push access and creates the ticket.
- Old per-user GitHub PAT path + write-key modal fully removed from board JS.

Superseded from original spec:
- Three board modes — collapsed to repo-visibility-driven.
- Write-key prompt + localStorage cache — replaced with GitHub token.

Closes the "Add Story button UX" memory note.
