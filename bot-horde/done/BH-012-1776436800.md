# [BH-012] Show active agent/assignee on board cards via live locks

- **Priority**: Medium
- **Effort**: S

- **Completed**: 2026-04-16

## Goal
Board cards should display which agent or person is working on a ticket — and once assigned, that name stays with the ticket for its entire life (in-progress, done, blocked, not-doing). Two layers: a **permanent assignment** baked into the ticket markdown, and a **live activity indicator** from the locks table.

## Why
Right now you can't tell who's working on what just by looking at the board. In a multi-agent setup where several Claude instances are working in parallel, this visibility is critical — it prevents duplicate work and lets the human see at a glance what's happening. And when a ticket is done, you should still see who did it.

## Done when

### Permanent assignment (the name that sticks forever)
- When a ticket moves to in-progress, the agent/person name is written to the `**Assigned to**` field in the ticket's markdown file (via a GitHub commit, same as cm-write)
- The 🦀 + name (e.g. "🦀 crabFather") shows on the card face AND in the expanded detail view
- The name persists through every status change — in-progress, done, blocked, not-doing. It never disappears.
- The `Assigned to` field in the markdown is the source of truth. The board renders it.

### Live activity indicator (who's actively working right now)
- On board load, query the `locks` table and overlay a "working now" pulse/glow on cards with active locks
- When a lock is acquired or released, the board updates in real time
- The live indicator is visual only (e.g. a pulsing border or animated 🦀) — distinct from the permanent name badge
- Cards with active locks stand out at a glance
- Works correctly when multiple agents are working simultaneously

### Agent identity
- Each agent/person picks their own name when they first enter their write key (stored in localStorage as `cm_agent_name`)
- The name appears as "🦀 crabFather" on the card — the crab icon is the universal agent marker

## Desired output
A board where every in-progress/done/blocked ticket shows 🦀 + the name of who worked on it, permanently. Tickets being actively worked on RIGHT NOW also pulse or glow.

## Success signals
- Move a ticket to in-progress → the agent name is written to the markdown file and shows on the card
- Ticket moves to done → the name is still there
- Opening the board while an agent is mid-task shows their name + a live activity pulse
- Lock release removes the pulse but NOT the name

## Failure signals
- Agent name disappearing when a ticket moves to done
- Stale live indicators that persist after an agent finishes
- Lock query failing silently and showing no indicators at all
- Performance degradation from polling the locks table too aggressively

## Tests
- Manual QA: move a ticket to in-progress → verify 🦀 + name appears on card and in detail
- Manual QA: move that ticket to done → verify the name is still there (permanent)
- Manual QA: while an agent is actively working, verify the live pulse/glow appears
- Manual QA: after agent finishes, verify pulse stops but name stays
- Verify `locks` table RLS allows anon SELECT (currently it doesn't — needs a policy addition)
- Verify `Assigned to` field in markdown file is updated when ticket moves to in-progress

## Notes
**RLS gap**: the `locks` table currently has no SELECT policy for anon/authenticated — it's invisible to the board's anon key. This ticket needs to add a read-only policy: `CREATE POLICY locks_select_all ON public.locks FOR SELECT TO anon, authenticated USING (true);` plus `GRANT SELECT ON public.locks TO anon, authenticated;`. This is safe because lock data (ticket_id + agent name + timestamp) is not sensitive.

**Real-time options**: (a) Supabase Realtime `postgres_changes` on the `locks` table (requires Realtime enabled on the table), or (b) broadcast events from the Edge Function that acquires/releases locks (if one exists), or (c) poll on an interval. Option (a) is cleanest if Realtime is enabled.

**Crab identity**: the agent/person picks their own name when entering their write key (stored as `cm_agent_name` in localStorage). The crab + name badge ("🦀 crabFather") sits in the top-right of the card face, visible without expanding. When a ticket moves to in-progress, the name is written to the markdown `Assigned to` field (permanent) AND to the `locks` table `agent` field (transient, for live indicator).

**Color coding**: each crab gets a deterministic accent color derived from its name (pick from a small palette of 8–10 distinct colors). The card's left border uses the same color so you can scan the board and instantly group work by agent.

**Two-layer display**: the permanent name badge is always visible once assigned (rendered from the `Assigned to` field in the markdown). The live pulse/glow is an overlay that appears only while a lock is active. When the agent finishes and releases the lock, the pulse stops but the name stays.
