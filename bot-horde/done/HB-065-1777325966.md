## Goal
Give adopters a way to know what version of the PM skill is installed and a one-liner to upgrade — instead of today's "delete the file and re-run setup."

## Why
The skill installs globally to `~/.claude/skills/product-manager/SKILL.md`, shared across all Horde of Bots projects on the user's machine. Adopters managing more than one project (or returning months later) can't tell whether their skill is current. There's no version, no upgrade command. A small `## Version: 1.0.0` line in SKILL.md plus a `--upgrade-skill` flag on setup.sh closes the gap.

## Done when
- `skills/product-manager/SKILL.md` (source) has a `## Version` line at the top with a semver value.
- `setup.sh` reads the version line from both the installed and the upstream copy. If they differ, it prints the diff (`local 1.0.0 → upstream 1.1.0`) and asks whether to upgrade.
- `bash setup.sh --upgrade-skill` (or `HORDEOFBOTS_UPGRADE_SKILL=yes`) bypasses the prompt and forces overwrite.
- The non-interactive path (HB-062) treats the upgrade prompt the same way as the migration prompt.
- Adopters running setup.sh today (without the version line) see the new version on first re-run after this lands; behavior is purely additive.

## Desired output
An adopter on a 6-month-old install runs setup.sh and learns "your PM skill is v1.0.0; upstream is v1.3.0. Upgrade? [y/N]." One command later they're current.

## Success signals
- Adopters with multiple Horde of Bots projects can tell which version is on their machine.
- "How do I upgrade" stops being asked.

## Failure signals
- Version drift between adopters becomes noisy if we bump versions for trivial changes. Mitigation: only bump when behavior changes.
- Adopter pinned a custom local edit to SKILL.md and the upgrade overwrites it. Mitigation: detect non-version edits via line count or hash and warn before overwrite.

## Tests
- Manual: install setup.sh fresh → SKILL.md has version line, no prompt.
- Manual: bump version in source repo → re-run setup.sh → upgrade prompt appears.
- Manual: `HORDEOFBOTS_UPGRADE_SKILL=yes bash setup.sh < /dev/null` → upgrade runs.

## Notes
- **Alternative considered**: per-project skill installs. Rejected — Claude Code's skill mechanism is global by design. Per-project would require its own loader.
- **Out of scope**: rolling back. Adopters re-install from a tag if they need an older version.
- **Confidence**: 80%.
