## Goal
Move every Horde of Bots artifact that currently sits at repo root (`BOTHORDE.md`, `build.sh`, `build_lib.py`, `bot-horde-board.html`, `bot-horde-config.json`) into the `bot-horde/` folder, so that the rule "anything under `bot-horde/` is dev tooling" becomes mechanically true and `.dockerignore` / `.gcloudignore` exclusions collapse to one line.

## Why
The current scattered layout forces every adopter and every LLM to independently build a mental model of which files belong to Horde of Bots and which are product. That mental model fails — see HB-048's underlying feedback. Consolidation makes the rule enforceable by tooling instead of tribal knowledge, and it's the enabler for HB-050's single-line deploy-ignore default.

## Done when
- `BOTHORDE.md` lives at `bot-horde/BOTHORDE.md`.
- `build.sh` lives at `bot-horde/build.sh`. It still runs from repo root (`bash bot-horde/build.sh`).
- `build_lib.py` lives at `bot-horde/build_lib.py`.
- Board is emitted at `bot-horde/board.html`.
- `bot-horde-config.json` lives at `bot-horde/config.json`.
- `setup.sh` writes the CLAUDE.md import as `@bot-horde/BOTHORDE.md`, and installs the new layout for fresh installs.
- `setup.sh` detects the legacy layout (root-level `BOTHORDE.md` + root-level `bot-horde-board.html`) and offers an explicit migration step that moves the five files and rewrites the CLAUDE.md import. Migration is opt-in, not forced silently, and is idempotent.
- `.github/workflows/build-board.yml` updated: `run: bash bot-horde/build.sh`, `git add bot-horde/board.html`.
- `tests/` updated so every test references the new paths. `test_build.py` integration test runs the script from its new location.
- README.md, SETUP.md, docs/onboarding/README.md, CLAUDE.md references all updated to new paths. No stale `bot-horde-board.html` or `bot-horde-config.json` references remain in live code/docs.
- Generated board HTML has no hard-coded references to the old filenames.

## Desired output
After this lands, `ls` at the repo root of an adopter shows `bot-horde/` and `setup.sh` — nothing else is Horde of Bots's. Adding `bot-horde/` to `.dockerignore` fully excludes Horde of Bots from every deploy artifact. A new contributor opens the repo and instantly knows which folder is workflow tooling.

## Success signals
- `.dockerignore` exclusion for Horde of Bots is one line (`bot-horde/`), verified in a sample adopter repo.
- No adopter reports "Horde of Bots files leaked into my deploy" post-ship.
- LLMs asked to summarize recent changes in an adopter repo cleanly bucket Horde of Bots vs product commits by folder.

## Failure signals
- Existing adopters' boards break silently because their workflow still references `build.sh` at root. Mitigation: setup.sh migration path + release note.
- `bash bot-horde/build.sh` breaks on Windows Git Bash paths. Mitigation: CI integration test.
- `build.sh` reads config by absolute path (e.g. `bot-horde-config.json` as sibling) and the move breaks credential loading.
- Adopters with bookmarked `bot-horde-board.html` URLs (GitHub Pages) hit a 404 until they update the link.

## Tests
- `tests/test_build.py`: run `bash bot-horde/build.sh` from repo root, confirm `bot-horde/board.html` is produced, confirm old `bot-horde-board.html` is NOT produced.
- Unit: `parse_ticket` and `parse_feature_set` still work unchanged (pure functions, just moved).
- Manual: on a throwaway clone with the legacy layout, run setup.sh migration path — verify it moves all five files correctly and rewrites the CLAUDE.md import. Run migration again — verify no-op.
- Manual: `grep -r "bot-horde-board.html" .` and `grep -r "bot-horde-config.json" .` after the change — the only hits should be historical ticket bodies and changelog notes.
- Manual: open the generated `bot-horde/board.html` in a browser and verify cards, Supabase live locks, and Add Story still function.

## Notes
- **Alternative considered**: keep the root layout but aggressively prefix files (`Horde of Bots-build.sh` etc.). Rejected — doesn't solve the one-line deploy-exclude problem and leaves the mental model fractured.
- **Alternative considered**: keep `BOTHORDE.md` at root, move the rest. Rejected — feedback explicitly calls out root-level BOTHORDE.md as part of the "looks like product" problem. Keeping it out of the folder breaks the one-folder rule that makes everything else mechanical.
- **Alternative considered**: emit the board as `bot-horde/index.html` so Pages serves it at `/bot-horde/`. Attractive but out of scope for this ticket — stick with `board.html` and let a future ticket decide the public URL story.
- **Breaking change**: yes. Existing adopters must run migration. Ship with a visible changelog entry and tested setup.sh migration path.
- **Out of scope**: deploy-ignore defaults (HB-050), opt-in CI commits (HB-051), any change to ticket format.
- **Risk (highest)**: `setup.sh` runs remotely via `curl | bash`. Migration branch must be idempotent and must not destroy local uncommitted changes to any of the five files.
- **Confidence**: 75%. Mechanically obvious, but the breaking-change cost is real.

## Resolution
Shipped in four phases:
1. File moves — `git mv` the 5 root files (`BOTHORDE.md`, `build.sh`, `build_lib.py`, `bot-horde-board.html` → `board.html`, `bot-horde-config.json` → `config.json`) into `bot-horde/`. Updated `build.sh` internals: `cd "$(dirname "$0")/.."` to reach repo root, `sys.path.insert(0, str(CM))` for build_lib import, config read from `CM / "config.json"`, output written to `bot-horde/board.html`.
2. CI/tests — `.github/workflows/build-board.yml` runs `bash bot-horde/build.sh` and stages `bot-horde/board.html`. `tests/test_build.py` copies build scripts into `tmp_path/bot-horde/` and asserts against the new output path. Full 77-test suite passes.
3. Docs — `README.md`, `SETUP.md`, `docs/onboarding/README.md`, project `CLAUDE.md`, and `bot-horde/BOTHORDE.md` all updated to reference new paths. Dead-referencing tickets in `not-doing/` and `done/` left alone per the "don't backfill historical tickets" rule.
4. setup.sh — added legacy-migration branch that detects root-level files, prompts `[y/N]`, moves all five to new paths, and rewrites `@BOTHORDE.md` → `@bot-horde/BOTHORDE.md` in `CLAUDE.md`. Idempotent (second run is a no-op). Fresh installs download `BOTHORDE.md` directly to `bot-horde/BOTHORDE.md` and write the new import line. Smoke-tested in a tmp dir: all files moved, CLAUDE.md rewritten, second run detected no legacy files.

**Known breaking change**: adopters on GitHub Pages using `your.github.io/repo/bot-horde-board.html` need to update their bookmark to `.../bot-horde/board.html`. Documented in SETUP.md.
