## Goal
Give each Kanban column a small animated SVG scene in its header showing a blacksmith at the matching stage of work — raw materials piled up for backlog, hammering on the anvil for in-progress, finished blades on a display rack for done, and so on.

## Why
The board today is visually flat — text cards in gray columns. A thematic animated scene per column makes work state feel visceral, rewards progress (watching cards move toward the "done" rack is satisfying), and gives Bot Horde a memorable visual identity that stands out from every other Kanban tool.

## Done when
- Each of the 5 Kanban columns (backlog / in-progress / done / blocked / not-doing) has a custom SVG scene at its header (~200×200px max)
- Scenes are animated with CSS keyframes only — no external animation libraries, no sprite sheets
- All five scenes share one cohesive visual style: forge / blacksmith workshop, line-art with muted fills
- Dark mode palette preserves the mood (cooler, dimmer, still readable)
- Animations remain smooth with 50+ tickets rendered across columns
- Scenes degrade gracefully: static SVG renders correctly even if CSS animations fail
- No per-card animation — cards remain compact and scannable

## Desired output
A visually distinctive board that immediately communicates "this is a workshop" — each column header is a small animated vignette of the blacksmith working in that state. Tickets flowing across the board feel like material moving through a real workflow.

## Success signals
- First-time visitors notice and comment on the scenes
- Animations loop smoothly at 60fps in Chrome, Safari, Firefox
- Dark mode still feels alive (not dead/flat)
- Board readability is unchanged or improved — users can still scan and click tickets without distraction
- The visual identity becomes a reason people share screenshots

## Failure signals
- Animations distract from reading tickets — users can't scan cards easily
- CPU spike with many cards rendered (animations repaint unnecessarily)
- Scenes look broken, ugly, or pixelated on small screens
- Scenes fail to render without JavaScript — should degrade to static SVG
- Dark mode palette makes scenes muddy or unreadable
- Scope creep into per-card animations, sprite sheets, or dependency-graph rendering

## Tests
- Manual visual check in Chrome, Safari, Firefox (light + dark mode)
- Manual responsive check at 560px, 900px, 1280px widths
- Performance check: render 100 tickets across columns, verify 60fps on a mid-range laptop
- Accessibility check: `prefers-reduced-motion` media query disables animation loops
- Build pipeline check: scenes are CSS-only (no JS dependency); board renders on file:// without a server
- Snapshot or visual regression test on the generated HTML (optional — flag if worth it)

## Notes
**Design agreed with user**:
- Theme **C (forge/blacksmith)** — matches the "build things" spirit of a dev tool; tree is generic, bird doesn't map to growth cleanly
- Ambition level **V1** — column-header SVG scenes, not per-card animation, not sprite sheets
- New feature set (feature-set-003-visual-language) — polish is going to be a recurring stream, keep workflow-hygiene clean

**Scene-by-scene sketch**:
- **Backlog**: pile of raw ore/ingots on the workshop floor, unlit forge in the background (static — nothing to animate, raw materials are just *there*)
- **In progress**: blacksmith at the anvil, hammer lifting and striking on a slow loop (~1.5s), sparks bursting at each strike, flames flickering in the forge
- **Done**: finished blade/gear on a display rack, slow light shimmer crossing the surface every 4–6s
- **Blocked**: forge gone cold, blacksmith sitting on a barrel scratching his head, tool mid-project on the anvil, slow "nope" head-shake every few seconds
- **Not-doing**: scrap bin outside the workshop with a discarded item tossed in, light dust puff settling on a loop

**Alternatives considered**:
- V0 (emoji + CSS pulses) — rejected; too thin to justify the effort, doesn't deliver the "wow"
- V2 (sprite sheet + card-move handoff animations) — deferred; ship V1 first, verify the concept lands, then decide whether V2 is worth the production cost
- Per-card animation — rejected for v1; clutters the card UX and hurts scan-readability
- Tree / plant theme — rejected; generic, doesn't match the dev-tool spirit
- Bird / egg theme — rejected; "growth" doesn't map onto the Kanban stages cleanly

**Follow-on candidates** (if V1 lands well):
- V2: pixel-art sprite sheets + card-move handoff animations ("blacksmith picks up the card, slides it onto the anvil")
- Feature-set aggregate view: show each feature-set as a "forge" with N forged items
- Staleness effect: in-progress card sitting for >7 days gets a subtle "aging" tint
- Sound effects on card-move events (optional, off by default)

**Sequencing**: opens feature-set-003. Low dependency on other work — could land any time. Defer until feature-set-001 (Supabase upgrade) is underway or done so mechanics work isn't delayed by polish work.

**Out of scope for this ticket**:
- Per-card animations (defer to V2)
- Sprite-sheet production (defer to V2)
- Sound design (defer)
- Theme switcher beyond light/dark (separate ticket if desired)

## Decision (2026-04-16)
Not doing, for now. After shipping feature-set-001 and living with the board, the forge animation direction felt like unnecessary scope — the live lock pulse + crab badges from BH-012 already carry the "this is alive" signal. Revisit only if a clear reason to add thematic motion emerges.
