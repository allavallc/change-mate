## Goal
Once polling (CM-057) and direct Add Story (CM-058) are live, remove every trace of Supabase from the codebase: the `supabase/` folder, the config fields, the verify scripts, the Team-path setup docs, the live lock pulse UI, and any tests covering Supabase pieces.

## Why
Leftover Supabase code is a drag on the project — bigger surface area for adopters to read, more ways setup can fail, more tests to maintain, more docs to keep in sync. The whole point of feature-set-005 is to be done with that backend. This ticket finishes the job.

## Done when
- `supabase/` folder is deleted (migrations, function, scripts, README).
- `change-mate/config.json` no longer requires `supabase_url` or `supabase_publishable_key`. Existing values are ignored.
- `tests/test_verify_supabase.py` deleted (or archived under historical notes).
- `scripts/verify_supabase.py` deleted.
- The board no longer has the "live" header indicator (or it's repurposed to show polling status — see CM-057).
- The board no longer has the lock-pulse `cm-active` class, the postgres_changes subscription, or any Realtime channel code.
- `SETUP.md` is rewritten — no more Solo / Team paths split. Single path: install → push → use. No CLI, no Edge Function, no migrations.
- `README.md` updated — drop the Supabase-related "Files" rows and the Live lock registry section (Gist version was already retired).
- `setup.sh` no longer references Supabase config keys. `auto_commit_board` mode-aware default rewires: no longer keyed off `supabase_url`. Suggested replacement: default `true` always, since the board needs to be committed to ship to Pages now that Realtime is gone.
- `change-mate/CHANGEMATE.md` Real-time sync section deleted, lock registry section deleted (or updated to explain the markdown-as-lock convention already implicit in the workflow).
- `.github/workflows/change-mate-rebuild-board.yml` no longer reads `SUPABASE_URL` / `SUPABASE_PUBLISHABLE_KEY` env vars.
- All tests pass. No dead imports, no stale references in docs.

## Desired output
An adopter pulls down the change-mate repo, runs `bash setup.sh`, commits the resulting `change-mate/` folder, and the board works on GitHub Pages within minutes — no second account, no CLI, no migrations. SETUP.md fits on one screen.

## Success signals
- New adopter can go from zero to working board in under 5 minutes.
- Code repo size shrinks (Supabase folder + tests removed).
- No more "is Supabase paused?" support questions.

## Failure signals
- A stale reference in docs sends an adopter chasing a Supabase setup that no longer applies.
- An existing adopter's config breaks because we removed required keys. Mitigation: ignore the old keys silently, don't fail on their presence.
- The `auto_commit_board` mode-aware default no longer makes sense without `supabase_url` as a signal — needs a different default rule (always-on is the simplest).

## Tests
- Existing pytest suite passes after deletions (clearly some test files will be deleted with the deletion of source files — that's expected).
- Manual: fresh clone → `bash setup.sh` → push → open Pages → confirm board works without ever opening Supabase.
- Manual: grep -r "supabase" . — only hits should be in done/historical tickets, never in live code.

## Notes
- **Alternative considered**: leave the Supabase code in place behind a config flag. Rejected — leftover dead code rots and confuses adopters.
- **Alternative considered**: keep the locks table for the live agent indicator. Rejected — drop the indicator entirely. The dashed-border + walking-robot already says "in progress" loud enough.
- **Out of scope**: redesigning the board look (separate ticket if pursued).
- **Migration risk**: existing adopters have Supabase secrets, RLS policies, and Edge Function deployed. We don't touch their Supabase project — they can leave it alone or delete it on their own. The change-mate code stops calling it; that's all.
- **Confidence**: 80%. Mostly mechanical deletions. The only design call is what to do with `auto_commit_board` mode-awareness; my recommendation is to default `true`.
