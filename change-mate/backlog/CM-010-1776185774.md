# [CM-010] Second-LLM review pass on ticket drafts before user sees them

- **Status**: open
- **Priority**: Low
- **Effort**: M
- **Feature set**: feature-set-002-workflow-hygiene
- **Assigned to**: 
- **Started**: 
- **Completed**: 

## Goal
Add an optional second-LLM review step that critiques a ticket draft (produced by the product-manager skill) before the draft is shown to the user — catching weak "done when" criteria, missing success/failure signals, or muddled goals.

## Why
The product-manager skill (CM-004) lets one LLM act as the PM and draft tickets autonomously. Drafts will occasionally be thin, vague, or self-consistent-but-wrong. A second LLM with a critic prompt — senior PM reviewer, or senior tech lead — can flag those issues before they reach the user, raising draft quality and cutting review rounds.

## Done when
- A configurable review step runs after the PM skill drafts a ticket
- Review output is either: "approved" (pass through) or "revise with: …" (draft rewritten once)
- Config flag in `change-mate-config.json` (e.g. `reviewer_enabled: true/false`, `reviewer_persona: "senior-pm" | "senior-tech-lead"`)
- Off by default; explicit opt-in
- User sees the final (reviewed) draft — the review exchange is not shown unless they ask

## Desired output
An optional critic pass that silently improves drafts.

## Success signals
- Turned on: drafts arrive with tighter acceptance criteria and clearer risk callouts
- Turned off: behavior is identical to CM-004 baseline

## Failure signals
- Latency spike on ticket creation (every draft now costs 2 LLM calls)
- Reviewer hallucinating requirements the PM never captured — draft drifts from user intent
- Infinite revise loops if not capped at one pass

## Tests
- Unit test: reviewer disabled → single draft pass-through unchanged
- Unit test: reviewer enabled + approved → draft unchanged
- Unit test: reviewer enabled + revise → draft rewritten exactly once, not looped

## Notes
**Sequencing**: depends on CM-004 (the PM skill must exist first). Low priority — ship only after the baseline PM skill has been in real use long enough to see where drafts actually get weak.

**Alternatives considered**:
- Always run reviewer (no toggle) → rejected; doubles cost and latency for every draft
- Self-review by the same LLM in a second prompt → rejected; same blind spots, less critical distance
