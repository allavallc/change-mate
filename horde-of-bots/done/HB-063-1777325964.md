## Goal
Wrap the `# Horde of Bots\n@horde-of-bots/HORDEOFBOTS.md` block setup.sh appends to `CLAUDE.md` with HTML comment markers so users can find it, understand what it does, and not accidentally delete it without realizing.

## Why
Today the import is two ambiguous lines. A user editing CLAUDE.md to tighten their own rules can innocently delete those lines and silently lose the Horde of Bots workflow — the agent stops following HORDEOFBOTS.md with no error. A `<!-- Horde of Bots import block — managed by setup.sh -->` marker pair makes the intent obvious and gives idempotency a stronger anchor.

## Done when
- setup.sh wraps the appended block:
  ```
  <!-- Horde of Bots import block — managed by setup.sh; remove the block to disable Horde of Bots -->
  # Horde of Bots
  @horde-of-bots/HORDEOFBOTS.md
  <!-- /Horde of Bots import block -->
  ```
- Idempotency check uses the opening marker as the existence test (replaces the current `grep -qF "$IMPORT_LINE"`).
- Migration path: when setup.sh detects an existing un-marked `@horde-of-bots/HORDEOFBOTS.md` line, it offers (or auto-on first re-run) to wrap it with markers — without duplicating the import.

## Desired output
A user opening CLAUDE.md sees what the block is for and either keeps it or removes the entire block deliberately. No silent footgun.

## Success signals
- Zero issue reports of "the agent stopped following the workflow and I don't know why."
- Adopters re-running setup.sh on an existing install do not get a second copy of the import.

## Failure signals
- The HTML comment syntax doesn't render cleanly in some markdown viewers. (It renders as nothing, which is fine.)
- A user removes only one of the two markers and breaks the block. Mitigation: check both markers exist together, treat one-without-other as a manual edit and don't auto-fix.

## Tests
- Manual: fresh install → CLAUDE.md has the wrapped block.
- Manual: re-run setup.sh → no duplicate markers, no duplicate import.
- Manual: remove the markers and re-run setup.sh → restores them around the existing import.

## Notes
- **Out of scope**: any other CLAUDE.md edits (other tooling can append its own blocks; we only manage ours).
- **Confidence**: 95%.
