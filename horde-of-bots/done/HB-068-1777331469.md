## Goal
Give bots and humans a deterministic way to answer "is my local copy of Horde of Bots up to date?" — without relying on file mtimes, hashes, or "delete and re-run." A single `horde-of-bots/MANIFEST.json` lists every managed file and its upstream version stamp. Local copy includes the manifest as of last install. Upgrade = fetch upstream manifest, diff, fetch the files that changed.

## Why
After HB-067 added the no-secrets rule, existing adopters had no path to the new HORDEOFBOTS.md / INSTALL-FAQ.md other than `delete + re-run` or manual curl. That same gap exists for every future doc/script update. A manifest closes the gap once: bots fetch one tiny JSON file, diff against their local copy, know exactly what's stale, and can update only what changed.

## Done when
- `horde-of-bots/MANIFEST.json` exists at repo root with `schema_version`, `updated` (ISO 8601 timestamp), and `files` (path → version date) for every managed file: HORDEOFBOTS.md, INSTALL-FAQ.md, build.sh, build_lib.py, config.json, the workflow YAML, and the PM skill.
- `setup.sh` installs MANIFEST.json on fresh install (idempotent — skipped if present).
- `HORDEOFBOTS_CHECK_UPDATES=yes bash setup.sh` (or `--check`) fetches the upstream manifest, diffs against local, prints one stale file per line, exits non-zero if anything is stale (CI-friendly).
- `HORDEOFBOTS_UPGRADE_DOCS=yes bash setup.sh` fetches the upstream manifest and re-fetches every file whose version differs from local. After successful fetches, the local manifest is updated to match upstream.
- When no local manifest exists (existing adopters' first run after this lands), upgrade mode treats every tracked file as stale and fetches all of them — bootstraps cleanly.
- INSTALL-FAQ.md gets a "How do I know if I need to update?" section pointing at the manifest + the two env vars.
- Manifest entries use simple ISO date strings (`2026-04-27`) — string comparison = chronological. Skill stays on semver since it already has `version: 1.0.0`.

## Resolution
Created `horde-of-bots/MANIFEST.json` with `schema_version`, `updated`, and a `files` map of every managed path → ISO date version (skill stays on its existing semver). Added two early-dispatch modes in `setup.sh`: `HORDEOFBOTS_CHECK_UPDATES=yes` fetches upstream manifest, diffs vs local via a small Python helper, prints stale entries and exits non-zero if anything is stale. `HORDEOFBOTS_UPGRADE_DOCS=yes` does the same diff and re-fetches every stale file, then overwrites the local manifest with upstream. When the local manifest doesn't exist (existing adopters bootstrapping), upgrade mode treats every tracked file as stale and fetches all of them. Fresh installs add MANIFEST.json to the standard download list. INSTALL-FAQ.md grew a "How do I know if I need to update?" section pointing at both env vars with copy-paste curl invocations.
