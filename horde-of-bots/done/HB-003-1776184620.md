## Goal
Let tickets reference other tickets so humans and agents can see dependencies and cross-ticket context at a glance.

## Why
Today "blocked" is just a folder — we lose *what* blocks *what*. Adding explicit relationships surfaces feature-set cohesion, reveals dependency chains, and makes the board useful beyond a flat list of cards.

## Done when
- Ticket format supports `Related`, `Blocks`, `Blocked by` fields (comma-separated CM-IDs) ✅
- `HORDEOFBOTS.md` documents how agents set and read these fields ✅
- Board renders relationships on the card (chips with cap + "+N more" overflow) ✅
- Board auto-infers the inverse of `Blocks` → `Blocked by` at render time (not persisted in files) ✅
- Moving a ticket to `blocked/` requires a `Blocked by` value ✅ (documented in HORDEOFBOTS.md)
- Round-trips cleanly through `build.sh` without data loss ✅

## Desired output
Three new optional fields in every ticket, rendered as chips on the card face, with the inverse edge inferred automatically. Build-time warnings for orphaned or cyclic references.

## Success signals
- New tickets created after this ships include the new fields where applicable
- Chips on CM-A say "Blocks: CM-B" and CM-B's card automatically shows "Blocked by: CM-A" without the user writing it twice
- Legacy tickets without these fields still parse and render
- Overflow displays cleanly on tickets with many relationships

## Failure signals
- Orphaned references (CM-XXX listed but file not found) — build logs a warning, chip is skipped
- Cycles (A blocks B blocks A) — build logs a warning, cards still render
- Board layout breaking when a ticket has many relationships — overflow cap prevents this
- Symmetric inference double-renders an edge (deduped in Python pre-render pass)

## Tests
- Parser unit tests: each field present / missing / single ID / multiple IDs / malformed / empty-value / whitespace / dedupe (7 tests in `tests/test_parser.py`)
- Build-time subprocess tests: orphan produces stderr warning + exit 0, cycle produces stderr warning + exit 0, inferred `blocked_by` appears in output HTML (3 tests in `tests/test_build.py`)

## Notes
**Design decisions agreed with user**:
- **Symmetry**: inferred at render time, not persisted. One side of the edge is written; renderer walks the reverse map and shows the counterpart automatically.
- **Placement**: all chips on card face, cap at 3 with "+N more" overflow; full list grouped by kind on card-detail open.
- **Orphan / cycle handling**: warn in build log (stderr), skip broken chip, keep building (non-fatal).
