## Goal
Remove three inaccurate or misleading claims from README.md and SETUP.md: (1) "no external dependencies beyond git" is false (Python 3 is required to run `build.sh`); (2) the README implies `bash horde-of-bots/build.sh` works after install but it doesn't yet (will be true once HB-061 lands); (3) GitHub Pages caveats — no warning that the board will be world-readable at `/horde-of-bots/board.html` when Pages is enabled on `main`.

## Why
A careful adopter reading the README before installing flagged each of these as a credibility issue. Inaccurate setup docs make people distrust the rest of the project. These are 30-minute copy edits that pay off in one place every adopter looks.

## Done when
- README.md "Contributing" or top-level intro states **Requirements: git + Python 3** (or removes the no-deps claim).
- README.md / SETUP.md `bash horde-of-bots/build.sh` mentions are accurate after HB-061 lands. (Pre-HB-061: add a note that the script ships in a follow-up.)
- SETUP.md Pages step adds a one-line warning: "If your repo is public, the board at `horde-of-bots/board.html` is world-readable. To keep it private, use a private repo + GitHub Pro."
- README adds a single line under the Setup section: "Tickets are committed to your repo. If your repo is public, your tickets are public."
- No invented features. Don't claim any capability the code doesn't actually have today.

## Desired output
An adopter reading the docs sees the actual requirements and constraints up front, decides if Horde of Bots fits, and doesn't get blocked by surprises after install.

## Success signals
- Adopter feedback shifts from "this didn't match what I expected" to "this matched the README."
- Public-repo / private-tickets confusion goes away.

## Failure signals
- Adopters get scared off by the warnings. Mitigation: keep the warnings short and factual — not a wall of caveats.

## Tests
- Manual: read README + SETUP cold, list any remaining inaccuracies.
- Manual: grep README for "no dependencies", "no external deps", "zero deps" — should be zero hits.

- **Out of scope**: changing how Pages works (we can't), changing the public-repo model (HB-066 documents workarounds).
