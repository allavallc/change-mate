## Goal
Move every Horde of Bots artifact that currently sits at repo root (`HORDEOFBOTS.md`, `build.sh`, `build_lib.py`, `horde-of-bots-board.html`, `horde-of-bots-config.json`) into the `horde-of-bots/` folder, so that the rule "anything under `horde-of-bots/` is dev tooling" becomes mechanically true and `.dockerignore` / `.gcloudignore` exclusions collapse to one line.

## Why
The current scattered layout forces every adopter and every LLM to independently build a mental model of which files belong to Horde of Bots and which are product. That mental model fails â€” see HB-048's underlying feedback. Consolidation makes the rule enforceable by tooling instead of tribal knowledge, and it's the enabler for HB-050's single-line deploy-ignore default.

## Done when
- `HORDEOFBOTS.md` lives at `horde-of-bots/HORDEOFBOTS.md`.
- `build.sh` lives at `horde-of-bots/build.sh`. It still runs from repo root (`bash horde-of-bots/build.sh`).
- `build_lib.py` lives at `horde-of-bots/build_lib.py`.
- Board is emitted at `horde-of-bots/board.html`.
- `horde-of-bots-config.json` lives at `horde-of-bots/config.json`.
- `setup.sh` writes the CLAUDE.md import as `@horde-of-bots/HORDEOFBOTS.md`, and installs the new layout for fresh installs.
- `setup.sh` detects the legacy layout (root-level `HORDEOFBOTS.md` + root-level `horde-of-bots-board.html`) and offers an explicit migration step that moves the five files and rewrites the CLAUDE.md import. Migration is opt-in, not forced silently, and is idempotent.
- `.github/workflows/build-board.yml` updated: `run: bash horde-of-bots/build.sh`, `git add horde-of-bots/board.html`.
- `tests/` updated so every test references the new paths. `test_build.py` integration test runs the script from its new location.
- README.md, SETUP.md, docs/onboarding/README.md, CLAUDE.md references all updated to new paths. No stale `horde-of-bots-board.html` or `horde-of-bots-config.json` references remain in live code/docs.
- Generated board HTML has no hard-coded references to the old filenames.

## Desired output
After this lands, `ls` at the repo root of an adopter shows `horde-of-bots/` and `setup.sh` â€” nothing else is Horde of Bots's. Adding `horde-of-bots/` to `.dockerignore` fully excludes Horde of Bots from every deploy artifact. A new contributor opens the repo and instantly knows which folder is workflow tooling.

## Success signals
- `.dockerignore` exclusion for Horde of Bots is one line (`horde-of-bots/`), verified in a sample adopter repo.
- No adopter reports "Horde of Bots files leaked into my deploy" post-ship.
- LLMs asked to summarize recent changes in an adopter repo cleanly bucket Horde of Bots vs product commits by folder.

## Failure signals
- Existing adopters' boards break silently because their workflow still references `build.sh` at root. Mitigation: setup.sh migration path + release note.
- `bash horde-of-bots/build.sh` breaks on Windows Git Bash paths. Mitigation: CI integration test.
- `build.sh` reads config by absolute path (e.g. `horde-of-bots-config.json` as sibling) and the move breaks credential loading.
- Adopters with bookmarked `horde-of-bots-board.html` URLs (GitHub Pages) hit a 404 until they update the link.

## Tests
- `tests/test_build.py`: run `bash horde-of-bots/build.sh` from repo root, confirm `horde-of-bots/board.html` is produced, confirm old `horde-of-bots-board.html` is NOT produced.
- Unit: `parse_ticket` and `parse_feature_set` still work unchanged (pure functions, just moved).
- Manual: on a throwaway clone with the legacy layout, run setup.sh migration path â€” verify it moves all five files correctly and rewrites the CLAUDE.md import. Run migration again â€” verify no-op.
- Manual: `grep -r "horde-of-bots-board.html" .` and `grep -r "horde-of-bots-config.json" .` after the change â€” the only hits should be historical ticket bodies and changelog notes.
- Manual: open the generated `horde-of-bots/board.html` in a browser and verify cards, Supabase live locks, and Add Story still function.

## Notes
- **Alternative considered**: keep the root layout but aggressively prefix files (`Horde of Bots-build.sh` etc.). Rejected â€” doesn't solve the one-line deploy-exclude problem and leaves the mental model fractured.
- **Alternative considered**: keep `HORDEOFBOTS.md` at root, move the rest. Rejected â€” feedback explicitly calls out root-level HORDEOFBOTS.md as part of the "looks like product" problem. Keeping it out of the folder breaks the one-folder rule that makes everything else mechanical.
- **Alternative considered**: emit the board as `horde-of-bots/index.html` so Pages serves it at `/horde-of-bots/`. Attractive but out of scope for this ticket â€” stick with `board.html` and let a future ticket decide the public URL story.
- **Breaking change**: yes. Existing adopters must run migration. Ship with a visible changelog entry and tested setup.sh migration path.
- **Out of scope**: deploy-ignore defaults (HB-050), opt-in CI commits (HB-051), any change to ticket format.
- **Risk (highest)**: `setup.sh` runs remotely via `curl | bash`. Migration branch must be idempotent and must not destroy local uncommitted changes to any of the five files.
- **Confidence**: 75%. Mechanically obvious, but the breaking-change cost is real.

## Resolution
Shipped in four phases:
1. File moves â€” `git mv` the 5 root files (`HORDEOFBOTS.md`, `build.sh`, `build_lib.py`, `horde-of-bots-board.html` â†’ `board.html`, `horde-of-bots-config.json` â†’ `config.json`) into `horde-of-bots/`. Updated `build.sh` internals: `cd "$(dirname "$0")/.."` to reach repo root, `sys.path.insert(0, str(CM))` for build_lib import, config read from `CM / "config.json"`, output written to `horde-of-bots/board.html`.
2. CI/tests â€” `.github/workflows/build-board.yml` runs `bash horde-of-bots/build.sh` and stages `horde-of-bots/board.html`. `tests/test_build.py` copies build scripts into `tmp_path/horde-of-bots/` and asserts against the new output path. Full 77-test suite passes.
3. Docs â€” `README.md`, `SETUP.md`, `docs/onboarding/README.md`, project `CLAUDE.md`, and `horde-of-bots/HORDEOFBOTS.md` all updated to reference new paths. Dead-referencing tickets in `not-doing/` and `done/` left alone per the "don't backfill historical tickets" rule.
4. setup.sh â€” added legacy-migration branch that detects root-level files, prompts `[y/N]`, moves all five to new paths, and rewrites `@HORDEOFBOTS.md` â†’ `@horde-of-bots/HORDEOFBOTS.md` in `CLAUDE.md`. Idempotent (second run is a no-op). Fresh installs download `HORDEOFBOTS.md` directly to `horde-of-bots/HORDEOFBOTS.md` and write the new import line. Smoke-tested in a tmp dir: all files moved, CLAUDE.md rewritten, second run detected no legacy files.

**Known breaking change**: adopters on GitHub Pages using `your.github.io/repo/horde-of-bots-board.html` need to update their bookmark to `.../horde-of-bots/board.html`. Documented in SETUP.md.
