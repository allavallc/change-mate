## Goal
Render staleness metadata on in-progress cards: how long since the ticket was claimed, and how long since the file was last committed.

## Why
A card sits in `in-progress/` looking alive even after the bot crashes. Today only Notes or git history reveal whether work is actually moving. Surfacing both timestamps on the card lets humans and reclaiming bots judge staleness at a glance — without adding a heartbeat field bots have to remember to update.

## Done when
- In-progress cards show two relative timestamps in the footer: `claimed Xh ago` (from `Started` field) and `last commit Yh ago` (from `git log -1 --format=%ct -- <ticket-file>`)
- Build script computes both at render time; no schema change
- If `last commit > stale_after_hours` AND `claimed > stale_after_hours`, card gets a dotted accent border to flag possible-stale
- `stale_after_hours` configurable in `config.json`, default 4
- Missing `Started`: omit the "claimed Xh" string, no error
- pytest: render with recent commit (no stale styling), with old commit (stale styling), with missing Started (no claimed display)

## Desired output
A user opens the board, sees one in-progress card flagged "claimed 18h ago · last commit 14h ago" with a dotted border, and immediately knows to check whether the bot is still alive.

## Success signals
- A stale claim gets reclaimed in the rollout window because someone noticed the flag
- Bots can read the same render output to decide whether to attempt reclaim

## Failure signals
- All cards trip the threshold because real work outpaces 4h — tune up
- Last-commit timestamp doesn't reflect work-in-progress (long-running task pre-commit) — acceptable; the heuristic is "are you visibly committing?", and bots that don't commit appear stale

## Tests
- Render with file commit >4h ago → stale border class in HTML
- Render with recent commit → no stale class
- Missing `Started` → no "claimed Xh" string in card

## Notes
- No new schema field. Render-only computation from existing data + git log.
- 4h threshold is a guess; configurable, no code change to tune.
- **Out of scope**: any reclaim automation. Render-only.
