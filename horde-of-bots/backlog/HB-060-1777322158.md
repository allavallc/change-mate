## Goal
Make `horde-of-bots/board.html` look like the SimplifyOps design system: dark editorial brutalism, single rust accent, hairline borders, mono-uppercase metadata, Big Shoulders Display headlines, JetBrains Mono labels, Inter body. Match `plan/style-guide.md` token-for-token where it applies.

## Why
The user owns `simplifyops.com` and wants the board to live in the same visual family. The current Inter-only friendly look feels disconnected from that brand. A unified aesthetic makes the board feel like a deliberate product, not a prototype.

## Done when
- Color tokens replace existing CSS variables: `--bg #0a0a0a`, `--bg-2 #111111`, `--ink #f2f1ee`, `--ink-dim #8a8680`, `--ink-dimmer #55514c`, `--line #1f1d1b`, `--accent #c4724a`. The old neutral palette and the green `#22c55e` status color are gone.
- Three Google Fonts loaded: Big Shoulders Display (500/700/900), Inter (300/400/500/600), JetBrains Mono (400/500). Existing Inter import replaced.
- Type roles applied: `--display` for column headers + ticket titles + page header; `--mono` for ticket IDs, priorities, efforts, dates, assignee names, agent crab badges; `--sans` for goal/why/notes prose.
- Cards: no rounded corners (`border-radius: 0`), no fills (transparent on `--bg`), 1px hairline `--line` borders, no `box-shadow`. Hover → border becomes `--ink-dim`.
- Status borders use dash pattern, not color:
  - Backlog: `1px solid var(--line)` (default)
  - In-progress: `1px dashed var(--accent)` (was green; robot recolors per-agent stays)
  - Done: `2px solid var(--accent)` (was green)
  - Blocked: `1px dotted var(--accent)` (was neutral)
  - Not-doing: existing strikethrough + opacity 0.4 unchanged
- Column headers become `.section-label` style: mono uppercase, `0.7rem`, `0.25em` tracking, `--accent`, prefixed with a 24px `--accent` rule. Format: `▬▬ BACKLOG · 03`.
- Card ID, priority, effort, dates: mono uppercase wide-tracked.
- Card title: Big Shoulders Display, 500–700 weight, uppercase, tight tracking.
- Card body sections (Goal, Why, etc) when expanded: Inter body, sentence case, `--ink-dim`.
- Header bar: `Simplify` style logo recoloured to `Horde of Bots` with `mate` in `--accent`. Backdrop-blur `rgba(10,10,10,0.75)` + `blur(14px)`. "+ Add story" button restyled as outlined CTA: 1px `--ink` border, mono uppercase, hover fills `--accent` with `--bg` text.
- Tabs (Board / Feature Sets): mono uppercase, 1px hairline border, active state uses `--ink` underline.
- Modal (Add story form): bare inputs with `border-bottom: 1px solid --line` only, no fills, no rounded corners. Labels above inputs in mono uppercase, `0.7rem`, `0.2em` tracking, `--ink-dim`. Submit becomes the outlined CTA pattern.
- Empty-column state ("No tickets") becomes mono uppercase `--ink-dimmer`.
- Toast notification: dark `--bg-2` with `--ink` text, 1px `--line` border, no rounded corners.
- Robot keeps its perimeter walk + per-agent color (intentional styleguide deviation, documented in Notes).
- 🦀 crab badge keeps its per-agent color (same intentional deviation).
- All transitions clamp to `0.2s–0.25s ease` per styleguide section 7.
- `prefers-reduced-motion` disables the robot walk and any other transitions.
- Selection: `::selection { background: var(--accent); color: var(--bg); }`.
- Pytest suite still passes (CSS swap shouldn't break tests; assertions on `cm-robot`, `cm-active-pulse`-removal, etc remain).

## Desired output
A user opens the board and immediately recognises it as part of the SimplifyOps brand — heavy display headlines on a near-black background, mono uppercase labels with rust accents, hairline grids. No friendly rounded corners, no green status colors, no soft drop-shadows. A walking colored robot on each in-progress card adds personality without breaking the brutalist frame.

## Success signals
- Side-by-side, the board reads as the same product family as simplifyops.com.
- Status is still readable at a glance via dash pattern (no color crutch needed).
- Per-agent identity (robot + crab colors) remains visible.

## Failure signals
- Status borders look too similar (solid 1px vs dashed 1px vs dotted 1px) at small sizes — may need to bump in-progress and blocked borders to 2px to keep the distinction crisp.
- Big Shoulders Display at column-header sizes overpowers the cards. Mitigation: column headers use `.section-label` mono pattern, not display.
- Rust accent on in-progress dashed border + per-agent-colored robot creates color clash. Mitigation: robot keeps per-agent color regardless; if visual clash is bad, escalate.
- Modal form looks too sparse without box borders.

## Notes
- **Intentional styleguide deviation #1**: per-agent crab and robot keep their 8-color palette. Bot-board needs visual agent identity; mono-only would erase the signal. Documented in feature-set-006 rationale.
- **Intentional styleguide deviation #2**: status uses `--accent` for done/in-progress/blocked. Styleguide says one accent total — this is consistent. The deviation from the styleguide spirit is using *dash pattern* as a status differentiator (not strictly mentioned in the guide, but stays true to "no introduced colors").
- **Out of scope**: floaters / circuit shapes drifting in the background. Could add later as a polish pass; skipping for v1 to keep the build small and focused.
- **Out of scope**: changing the column count, the card layout structure, or the Add Story form fields themselves. This is purely visual.
- **Risk**: dropping `prefers-color-scheme: dark` and the light-mode fallback. The site is dark-only by design per styleguide section 1. Acceptable. If a user's OS forces light, they get dark anyway — that matches simplifyops.com.
- **Confidence**: 80%. The styleguide is precise; mostly mechanical CSS substitution. Main risk is column-header presentation.
